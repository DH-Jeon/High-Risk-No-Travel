# 하리노트 (HARI-NOTE: High Risk, No Travel)

> 강원 가족 여행자를 위한 관광지 안전 리스크 점수화 + 안전 대체 코스 추천 웹서비스
> 『2026 관광데이터 활용 공모전』 ① 웹·앱 개발 부문 출품작 — 팀 새벽코딩

## 한 줄 소개

기존 여행 앱이 "강원 어디가 예뻐요"를 말한다면, 하리노트는 **"오늘 이 강원 관광지는 가족 여행에 주의 요인이 높으니, 비슷한 만족도의 더 안전한 대체지로 바꾸세요"**를 말합니다.

## 핵심 기능

1. **안전 점수 (0~100)** — TourAPI 강원 관광지별 폭염·강수·산불·미세먼지·의료·이동 위험을 공공기관 공식 임계값 기반 가중합으로 점수화
2. **위험 요인 분해** — "왜 주의해야 하는가"를 막대그래프로 시각화
3. **안전 대체 코스** — 거리·카테고리 유사도·점수·실내외 적합도 기반 대체 관광지 + 음식점 + 문화시설 반나절 코스
4. **가족·시니어 모드** — 아이/부모님 동반, 자차/뚜벅이별 위험 가중치 차등 적용
5. **출발 전 체크 리포트** — 산악기상·산불·통제·응급의료 한 장 요약

## 저장소 구조

```
├─ docs/        # 공고문, 제안서, 활동계획서, 기획 문서, ADR
└─ harinote/    # Next.js 웹 앱
```

## 실행 방법

```bash
cd harinote
pnpm install
pnpm dev        # http://localhost:3000
pnpm test       # 안전 점수 엔진 단위 테스트
```

기본값은 `DATA_SOURCE=mock`(실존 강원 관광지 fixture)로 API 키 없이 동작합니다.
TourAPI 키 발급 후 `harinote/.env.local`에 키를 넣고 `DATA_SOURCE=live`로 전환하면 실데이터를 사용합니다. 자세한 내용은 `docs/API키_발급_가이드.md` 참고.

## 데이터 파이프라인

```
TourAPI(KorService2) 강원 관광지 → 외부 공공데이터 결합(기상청·산림청·AirKorea·복지부·행안부)
→ 위험 정량화 → SafetyScore 산출 → 위험 유형 분류 → 대체지 매칭 → 화면 출력
```

## 기술 스택

- **Frontend**: Next.js (App Router, TypeScript, Tailwind CSS)
- **Backend/DB**: Supabase (PostgreSQL)
- **배포**: Vercel
- **지도**: 카카오 지도 API
- **데이터**: 한국관광공사 TourAPI 4.0 + 기상청 + AirKorea + 산림청 + 행정안전부 + 보건복지부
