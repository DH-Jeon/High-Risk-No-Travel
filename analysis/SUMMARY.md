## 2. 분석 파이프라인 (analysis/ 폴더, 01→10 순서 재현 가능)

1. **EDA** — 관광지 775건 중 envType 휴리스틱이 57%(443건)를 "일반 야외"로 뭉뚱그림.
   그 안에 표고 400m+ 99건 (산악 분류 55건보다 많음) → 분류 모델의 정량 근거
2. **데이터 수집** — 표고 2,086곳(SRTM 30m), 공식 서비스분류코드 v4.2(공모전 배포
   Drive, cat3 100% 매칭), 강원 119 구조출동 6,176건(소방안전 빅데이터 플랫폼)
3. **점수 엔진 Python 포팅** — 원본 vitest 경계 속성 20/20 재현 (`safety_engine.py`)
4. **클러스터링** — 1차 실패(극한 시나리오가 감점 상한 clamp에서 가중 신호를 지움)
   → 2차: 특보 직전 중간 강도 + 강수/강풍 분리 + cat2 블록 가중 축소.
   **k=6 채택 (bootstrap ARI 0.921)**
5. **외부 검증(119 데이터)** — 자연장소 사고는 구조가 유의하게 느림(시군 내 쌍대
   +3분, p=0.0008), 수난사고 분포 ↔ 수변·해안 관광지 분포 rho=0.49 p=0.038(탐색적).
   고지·오지형 ↔ 시군 구조시간은 사고 좌표 부재로 판정 불가
6. **분류기(RandomForest)** — 원천 피처만으로 CV acc **0.895** / macro-F1 0.870
   (envType 규칙 베이스라인 0.830 / 0.804). 규칙 오류 132건 중 116건 회수.
   중요도 1위 = 표고 → 성능 향상은 규칙이 안 쓰는 데이터에서 나옴
7. **산식 적용 영향 분석** — envType 보정(고지오지 & 400m+ → mountain, 93건):
   산불철 시나리오에서 93곳 −5점, **34곳 low→moderate 등급 변화**. 비대상 682곳 무변동
