# 기획: "그날 가도 될까" — 여행 날짜 기반 안전 점수 (합의용 1장)

- 작성: 박현준, 2026-07-09
- 상태: **팀 합의 대기** (동현 확인 후 착수)

## 1. 뭘 바꾸자는 건가

지금은 "**오늘** 이 관광지 가도 될까"만 답한다. 여행은 미리 계획하므로,
**"다음주 수~금에 가도 될까"**에 답하게 만들자.

- 사용자: 날짜(또는 기간)를 고르면 → 그 날짜 기준 안전 점수·요인·대체지·코스
- 점수 엔진(산식)은 그대로 — **입력(RiskInput)을 날짜에 맞는 값으로 채우는 층**만 추가

## 2. 날짜 구간별 데이터 3층 (핵심 설계)

| 여행일까지 | 입력 소스 | 필요한 작업 |
|---|---|---|
| D+0~3 | 기상청 **단기예보** (현재 사용 중인 getVilageFcst가 +3일까지 제공) | 코드가 오늘 값만 쓰는 부분을 날짜 파라미터화 |
| D+4~10 | 기상청 **중기예보** (중기육상 rnSt + 중기기온 taMin/taMax) | API 활용신청(기존 키) + 어댑터 1개 |
| D+11~ | **계절 모드**: 기후 평년값(30년, 시군·월별) + 계절 위험 prior | 평년값 수집(1회) + prior 설계 |

계절 prior 재료 (이미 확보/공식 기준):
- 산불: 산림청 **산불조심기간** (봄 2/1~5/15 · 가을 11/1~12/15) — 공식 기간 규칙
- 한파: 겨울 축 (기상청 한파특보 기준 + 기온감률×표고) — analysis/12 완성분
- 실증 근거: 강원 119 사고 월별 패턴 (12월 최다) — analysis/data 확보분

## 3. 역할 분담 제안

| 파트 | 담당 | 내용 |
|---|---|---|
| 중기예보 어댑터 (kma-mid.ts) | **동현** | 기존 kma.ts와 같은 패턴 (시군 대표점 + 캐시 + 폴백) |
| 단기예보 날짜 파라미터화 | 동현 (or 협의) | fetchKmaDailyWeather(nx, ny, **date**) |
| 기후 평년값 수집·내장 | **현준** | 시군×월 기온·강수 JSON (병원 데이터와 같은 내장 방식) |
| 계절 위험 prior 설계·검증 | **현준** | 분석 파트 — 119 데이터로 근거 제시 |
| 날짜 선택 UI + 날짜별 화면 | **현준** | 프론트 담당 영역 |

## 4. 단계 (작게 시작)

1. **Phase 1**: 상세 페이지에 날짜 선택(D+3까지) — 신규 API 없음, 리스크 최소
2. **Phase 2**: 중기예보 연동 → D+10까지
3. **Phase 3**: 계절 모드 → 어떤 날짜든 답함 (+ 코스/리포트에 기간 확장)

## 6.5 [7/9 진행] 계절 모드 설계 확정 — "통상일/궂은날" 분위수 시나리오

Phase 3 재료·설계는 분석 파트에서 선행 완료 (analysis/15~16b):
- **수집**: 강원 시군 대표점 30년(1991~2020) 일별 기후 → 시군×월 평년 테이블
  (Open-Meteo ERA5 — 기상청 공식 평년값 키 확보 시 교차검증 예정)
- **설계 반복**: v1(특보일 확률 기대 감점)은 확률 희석으로 장마철 계곡이 무위험으로
  나와 폐기 → **v2: 분위수 시나리오** — 그 달의 "통상일(중앙값)"과 "궂은날(90분위)"을
  점수 엔진에 그대로 입력해 **점수 범위**로 제시
- 예시 (통상/궂은날): 미산계곡 1월 81/44 · 7월 81/66, 계방산 1월 72/50
- UI 제안: D+11 이후 날짜 선택 시 "통상 81점 · 궂은날 66점 (30년 기후 기준)" 범위 표시
- 산불 달력(산불조심기간)·한파(표고 보정)는 시나리오에 내장됨

## 7. 리스크

- 중기예보는 하늘상태·강수확률 중심(강수량 없음) → 중기 구간은 점수 정밀도가
  단기보다 낮음을 UI에 명시 ("예보 기반 추정" 라벨)
