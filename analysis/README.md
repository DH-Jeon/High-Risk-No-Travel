# analysis/ — 위험 유형 분류 모델 (담당: 박현준)

> **중요: 이 폴더는 서비스 코드(`harinote/`)를 변경하지 않는다.**
> 점수 산식·엔진은 그대로 두고, 데이터 분석으로 산식의 사각지대를 찾아
> "적용하면 무엇이 바뀌는지"까지 검증한 제안 패키지다.

## 왜 했나

- 제안서 2단계 약속: "관광지 위험 유형 분류 — 규칙 기반 → ML 전환"
- 심사 예상질문 "AI가 정말 필요한가? 규칙 기반 필터링 아닌가?"에 대한 답 마련
- EDA 결과: 현행 envType 휴리스틱은 관광지 775건 중 **57%(443건)를 outdoor_general로 뭉뚱그림** — 이 안에 사찰·산촌체험마을 등 실질 산악·오지가 섞여 있음

## 핵심 발견: 6개 위험 유형, 그중 "고지·오지형"

| 유형 | n | 특징 |
|---|---|---|
| 근교전천후 | 312 | 저지대·의료 양호 |
| **고지·오지형 ★** | **132** | **평균 표고 536m·응급실 19.5km — 현행 규칙은 전부 "일반 야외" 취급** |
| 산악 | 123 | 산불·강풍 가중 (현행 규칙과 일치) |
| 해안 | 100 | 강풍 가중 (일치) |
| 수변 | 91 | 호우 가중 (일치) |
| 실내 | 17 | 기상 영향 낮음 (일치) |

4개 유형은 현행 규칙과 수렴(규칙 타당성 확인), 고지·오지형 132건(17%)이 신규 발견.

## 외부 검증 (강원 119 데이터)

- 자연장소 사고는 구조가 유의하게 느림: 시군 내 쌍대 비교 **+3분, p=0.0008** → 의료 접근성 감점 축의 실증 근거
- 수난사고 시군 분포 ↔ 수변·해안 관광지 분포 **rho=0.49, p=0.038** (탐색적 — 40건 소표본)
- 고지·오지형 ↔ 시군 구조시간: 판정 불가 (사고 데이터에 좌표 없음 — 시군 집계 한계)

## 서비스 적용 방안 (침습도 순)

1. **무변경 — 유형 배지**: `data/risk_types.json`(contentId→유형 775건)을 읽어 상세 페이지에
   "⛰️ 고지·오지형 — 응급의료 원거리" 배지 표시. 부모님 동반 프로필과 시너지
2. **최소 변경 — envType 보정**: 고지·오지형 & 표고 400m+ & general → mountain (93건).
   산불철 시나리오에서 −5점, 34건이 low→moderate로 — 영향 명세는 `data/envtype_fix_impact.csv`
3. **확장 — 분류기 이식**: `risk_type_classifier.joblib` — 신규 관광지·타지역(제주 등)에
   시나리오 계산 없이 유형+확률 할당 (이식 시 lat/lng 피처 제외 후 재학습 필요)

## 한계 (발표 때 선제 서술)

- 분류기 타깃은 자체 클러스터링 라벨 — "독립 정답 예측"이 아니라 "유형 체계의 이식 가능한 압축"
- 실내 유형 recall 0.588 (17건 소표본) → 실내는 기존 규칙 우선, 모델은 보완 (하이브리드)
- 수난 검증은 다중검정 보정 시 경계선 — "탐색적"으로 표현할 것
